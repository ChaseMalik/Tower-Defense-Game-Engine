20 /**
21  * Utility for supporting multiple languages in the view.
22  * The utility works by reading a list of resource files for different languages and
23  * uses JavaFX ObjectProperties (which use native JavaFX Observer/Observable pattern)
24  * to bind JavaFX nodes to certain texts. Furthermore, switching between languages is
25  * supported by a call to the setLanguage(String languageName) method. For any JavaFX
26  * node to update it's text it requires a 1-time setup to bind the property
27  * e.g.,
28  * myButton.getTextProperty().bind(MultiLanguageUtility.newInstance().getString("BUTTON_LABEL"));
29  * 
30  * public API:
31  * getInstance() --> instance of the utility
32  * initLanguages(String ... resourceFiles) --> initializes the languages in the utility
33  * setLanguage(String language) --> sets the language of the utility
34  * getCurrentLanguage() --> returns the current language
35  * getSupportedLanguages() --> ObjectProperty holding ObservableList for supported languages
36  * getStringProperty(String propertyName) --> gets the string property (ObjectProperty holding
37  * String)
38  * getStringListProperty(String propertyName) --> gets the string list property (ObjectProperty
39  * holding ObservableList)
40  * Requires a resource (.properties) file for each language.
41  * The name of the language must be specified in the resource file as a tag "language=..."
42  * If it is not, then the name of the language is assumed to be the name of the file.
43  * 
44  * Singleton pattern: 1 MultiLanguageUtility instance per program to handle language
45  * (assumes that a program can only have 1 language at any given time)
46  * 
47  * @author Jonathan Tseng
48  *
49  */
